How to View Your Landing Page

A complete guide to accessing your GoGetter landing page in different scenarios

Viewing Your Page Locally

If you have the HTML file on your computer:

1

Find the HTML file

Locate the index.html file on your computer (where you saved it).

2

Open with a browser

Right-click the file and select "Open with" then choose your web browser (Chrome, Firefox, Edge, etc.).

3

View your page

Your landing page will open in the browser and you can view it locally.

Note: When viewing locally, some features that require a server (like certain JavaScript functions) may not work perfectly.

Viewing on GitHub Pages

If you've deployed to GitHub Pages:

1

Get your GitHub Pages URL

Your GitHub Pages URL follows this pattern:

https://
username.github.io/repository-name
2

Open in browser

Enter this URL in your web browser's address bar and press Enter.

3

Bookmark for easy access

Once loaded, bookmark the page for future easy access.

Tip: Replace "username" with your GitHub username and "repository-name" with your repository name.

Viewing with Your Custom Domain

If you've set up a custom domain (www.gogetter.site):

1

Enter your domain

Open your web browser and enter your domain name:

https://
www.gogetter.site
2

Wait for DNS propagation

If it doesn't load immediately, wait a few hours for DNS changes to propagate.

3

Check GitHub Pages settings

Ensure your custom domain is correctly set in GitHub repository settings.

Note: Your custom domain is currently showing "DNS Check in Progress". This might take up to 24 hours to complete.

Check Your Page Status

Test if your page is accessible at different URLs:

Status: Checking...

We're testing the URL availability. This might take a moment.

Troubleshooting

Page not loading?

If your page isn't loading, try these solutions:

1

Check your repository settings

Ensure GitHub Pages is enabled for your repository in the Settings tab.

2

Verify file structure

Your HTML file must be named index.html and placed in the root of your repository.

3

Clear your browser cache

Sometimes old versions of pages are cached. Clear your browser history and cache.

4

Check for typos

Ensure there are no typos in your GitHub username, repository name, or custom domain.

Need more help? Contact GitHub Support or consult the GitHub Pages documentation.
