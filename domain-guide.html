Fix Domain Redirect Issue

Solving the problem where your domain redirects to Domains.co.za instead of your landing page

Why This Happens

When your domain redirects to your hosting provider (Domains.co.za) instead of your GitHub Pages site, it's usually because of one of these reasons:

  • DNS records are not correctly configured
  • The domain is still pointing to default parking pages
  • Nameservers are not properly set
  • There's a redirect set up in your domain control panel

Step-by-Step Solution

1

Log into Domains.co.za

Access your Domains.co.za account and navigate to the DNS management section for your domain (www.gogetter.site).

2

Remove Default Records

Delete any existing A records or CNAME records that might be pointing to Domains.co.za servers or parking pages.

3

Add Correct GitHub Pages Records

Create these DNS records exactly as shown:

Type Name Value TTL
CNAME www jeagergo.github.io 3600
A @ 185.199.108.153 3600
A @ 185.199.109.153 3600
A @ 185.199.110.153 3600
A @ 185.199.111.153 3600
4

Check Nameservers

Ensure your nameservers are set to Domains.co.za's nameservers (unless you're using custom nameservers):

  • ns1.domains.co.za
  • ns2.domains.co.za
5

Wait for Propagation

DNS changes can take up to 24-48 hours to fully propagate across the internet. Be patient during this process.

6

Verify in GitHub

In your GitHub repository, go to Settings > Pages and ensure your custom domain is correctly entered as "www.gogetter.site".

Quick Check While Waiting

While waiting for DNS changes to propagate, you can:

1

Test Using GitHub URL

Access your site directly via GitHub Pages:

https://
jeagergo.github.io/gogetter-landing/
2

Clear Your Browser Cache

Sometimes your browser caches old DNS information. Clear your cache or try accessing your site in an incognito window.

3

Check DNS Propagation

Use a tool like DNS Checker to see if your DNS changes have propagated globally.

Still Having Issues?

If after 48 hours your domain still redirects to Domains.co.za:

  • Double-check that you've removed all default records from Domains.co.za
  • Contact Domains.co.za support and ask them to ensure no server-side redirects are active
  • Verify that your domain is not set to "Parked" mode in your control panel

You might need to contact Domains.co.za support directly and ask them to:

  1. Remove any default parking page settings
  2. Confirm that your DNS records are properly configured
  3. Ensure there are no hidden redirects applied to your domain

Need more help? Contact Domains.co.za support or consult the GitHub Pages documentation.
