GitHub Pages Domain Setup

A step-by-step guide to configuring your custom domain with GitHub Pages

Understanding the DNS Check Process

When you add a custom domain to GitHub Pages, it goes through a verification process that can take anywhere from a few minutes to up to 24 hours.

⏳

DNS Check in Progress

Your domain www.gogetter.site is currently being verified by GitHub. This process checks that your DNS records are correctly configured.

Note: It's normal for this process to take some time. DNS changes need to propagate across the internet, which can cause delays.

Step-by-Step Configuration Guide

1

GitHub Pages Settings

In your GitHub repository, go to Settings > Pages. In the "Custom domain" section, enter your domain (www.gogetter.site) and click Save.

2

DNS Configuration at Domains.co.za

Log in to your Domains.co.za account and navigate to DNS management for your domain.

You need to create the following records:

Type Name Value TTL
CNAME www jeagergo.github.io 3600
A @ 185.199.108.153 3600
A @ 185.199.109.153 3600
A @ 185.199.110.153 3600
A @ 185.199.111.153 3600
3

Wait for DNS Propagation

After updating your DNS records, it can take up to 24 hours for the changes to propagate globally. GitHub will automatically detect when your DNS is properly configured.

Tip: You can use tools like DNS Checker to monitor the propagation status.

4

Enforce HTTPS (Recommended)

Once your domain is properly configured, go back to your GitHub Pages settings and enable the "Enforce HTTPS" option. This provides secure connections for your visitors.

Troubleshooting Common Issues

DNS Check Taking Too Long

If the DNS check has been ongoing for more than 24 hours:

  • Double-check that your DNS records are exactly as specified
  • Ensure there are no typos in your domain name in GitHub settings
  • Try removing the custom domain from GitHub, waiting 5 minutes, and adding it again

Domain Not Redirecting Properly

If your domain isn't working as expected:

  • Verify that you've configured both the CNAME and A records correctly
  • Check if your domain registrar has any additional requirements for custom domains
  • Ensure your GitHub repository is public (required for GitHub Pages)

Important: Avoid using a custom domain that starts with http:// or https:// in the GitHub settings. Just enter the domain name itself (e.g., www.gogetter.site).
